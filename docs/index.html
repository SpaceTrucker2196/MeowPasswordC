gameplay loop

Five candidates. One high-score winner.

meowpass pulls cat names from its embedded dictionary, adds capitals, numbers, and symbols, scores each candidate with several complexity heuristics, and keeps the least predictable one. Same CLI. Same security goals. New cartridge-shelf presentation.

▣

16,913 embedded cats

The full dictionary ships inside the binary like ROM. No extra data files. No network fetches.

✦

OS-grade randomness

getrandom() or /dev/urandom provide the bytes, with rejection sampling to avoid bias.

▲

Best-of-five selection

Each round generates five phrase passwords, then picks the one with the toughest score profile.

■

Quiet copy mode

--psssst copies the winner without printing it — stealth mode for the score chasers.

◆

Analyze any string

--analyze "foo" runs the same complexity board on a password you already use.

☼

One binary, no deps

Pure C11. Build with make or CMake. Clipboard support stays optional and system-native.

strategy guide

Pawthropology of complexity.

Under the hood, meowpass scores each candidate with four real metrics — Shannon entropy, compression resistance, pattern uniqueness, and character diversity — while the CLI keeps the catified language that makes the whole thing feel like a lost manual from a better timeline.

Tail Size
password length

Longer strings give the cat burglar less room to get lucky.

Ball of Yarn Entropy
shannon entropy (bits/char)

Tracks how unpredictable each character is. Messier yarn is harder to untangle.

Mashing Resistance
compression-ratio approximation

Passwords that compress well are repetitive. Repetitive passwords are low-score cartridges.

Shiny Foil Ball Uniqueness
pattern complexity

Distinct substring coverage reveals how varied the patterning is across the full phrase.

Organic NonGMO Catnip
character-class diversity

Measures the mix of lowercase, uppercase, digits, and symbols in the final password.

Overall Relavency
composite score · lower is better

The combined scoreboard. Low relevance means even a crafty cat has to keep feeding quarters.

The verdicts

< 2.0
Purrfect — a cabinet-top score that keeps the alley cats away.
< 4.0
Solid run — good whisker resistance with room to rack up more points.
< 6.0
Shaky quarter — a clever cat might still sneak onto the leaderboard.
≥ 6.0
Game over — even the kitten on player two can crack this one.

control panel

Flags & recipes.

# default — generate, score, show, copy hint
meowpass

# tune the recipe
meowpass --numbers 4 --symbols 3 --max-length 30

# silent — copy the winner to the clipboard, print nothing
meowpass --psssst

# audit a string you already have
meowpass --analyze "MyP@ssw0rd!"

# run the test suite
meowpass --test

# check GitHub for a newer release and install it
meowpass --update

insert cartridge

Two ways in.

C11. No external dependencies. Builds on Linux and macOS. Clipboard support uses xclip or wl-copy on Linux and pbcopy on macOS.

Make

git clone https://github.com/SpaceTrucker2196/MeowPasswordC.git
cd MeowPasswordC
make
./meowpass

CMake

cmake -B build
cmake --build build
./build/meowpass

Install system-wide

sudo make install
